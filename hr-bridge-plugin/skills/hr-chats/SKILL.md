# HR Bridge — резюме, чаты, статусы, Telegram

## Область применения

Правило действует **для всех чатов и коммуникаций по работе с кандидатами**: обе платформы
(**Rabota.by и HH.ru**), **все вакансии**, все каналы (`send_message`, `change_status`,
`send_telegram_message`). Это область действия, а не команда — фактические действия только
по явному запросу (см. «Правила»).

## Инструменты

- **list_responses(vacancy_id, provider)** — список откликов (id, resume_id, имя)
- **get_resume(resume_id, provider)** — полное резюме: опыт, навыки, контакты (email, телефон, telegram)
- **read_messages(nid, provider)** — переписка с кандидатом
- **send_message(nid, text, provider)** — написать в чат HH.ru / Rabota.by
- **change_status(nid, action, message, provider)** — сменить статус отклика (включая «Отказ»)
- **send_telegram_message(chat_id, text, bot_token)** — написать в Telegram через бота
- **list_vacancies(employer_id, provider)** — вакансии
- **get_me / token_status** — диагностика

## Смена статуса отклика

`change_status(nid, action, message="", provider="rabota")` переводит отклик между папками
работодателя. Допустимые `action`:

| action | Значение |
|--------|----------|
| `discard_by_employer` | **Отказ** |
| `consider` | Рассмотреть / в работу |
| `phone_interview` | Телефонное интервью |
| `interview` | Интервью |
| `assessment` | Тестовое / оценка |
| `offer` | Оффер |
| `hired` | Нанят |
| `discard_no_interaction` | Отказ (нет отклика кандидата) |
| `discard_vacancy_closed` | Отказ (вакансия закрыта) |

`message` (необязательно) уходит кандидату вместе со сменой статуса.

## Правила (постоянные)

- **Область:** правило распространяется на все вакансии и оба сайта (Rabota.by + HH.ru).
- **Согласие получено:** механизм смены статуса согласован владельцем. Не объяснять каждый
  раз, что это возможно, и не переспрашивать про сам подход — просто выполнять по запросу.
- `send_message` и `change_status` — **только по явному запросу** пользователя.
- **Смена статуса необратима из интеграции** (откат — вручную в интерфейсе сайта).
  - Единичное действие (1 кандидат) по явному запросу — выполнять сразу.
  - Массовое действие (группа/фильтр) — сначала показать список целей + текст (dry-run),
    применить только после «ок». Это одно подтверждение на всю пачку.
  - Перед массовым запуском убедиться, что выбрана правильная вакансия.
- **Никогда не выводить токены** (OAuth, bot_token) в лог/чат.
- Шаблон отказа по умолчанию (редактируемый): «Спасибо за отклик и уделённое время. Сейчас
  мы продолжаем с другими кандидатами; будем рады вернуться при подходящей вакансии.»
